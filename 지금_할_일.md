# 지금 단계에서 할 일 (순서대로)

---

## 1단계: 앱 실행해서 확인

1. 터미널에서 프로젝트 폴더로 이동
2. 아래 명령어 입력 후 엔터

```bash
npm run dev
```

3. 브라우저에서 `http://localhost:3000` 열기  
4. 로그인하고 QR 스캔 → 로그 한 번 남겨보기  
   → 저장 후 **QR 스캔 화면으로 돌아가는지**, **자주 쓰는 문구**가 보이는지 확인

→ **여기까지 되면** 1) 로그 후 초기화, 2) QR 문구 정렬, 4) 자주 쓰는 문구는 **이미 적용된 상태**예요. 따로 설정할 건 없어요.

---

## 2단계: (선택) 로그에 사진 넣고 싶을 때만

**사진 없이 글만 로그**해도 앱은 잘 돌아갑니다.  
**사진도 올리게** 하고 싶을 때만 아래를 하면 됩니다.

### 2-1. Supabase 들어가기

- [supabase.com](https://supabase.com) 로그인
- 사용 중인 프로젝트 선택

### 2-2. 로그 테이블에 “사진 주소” 칸 추가

1. 왼쪽 메뉴에서 **SQL Editor** 클릭
2. **New query** 로 새 쿼리 창 열기
3. 아래 문장 **그대로** 붙여넣기

```sql
ALTER TABLE logs ADD COLUMN IF NOT EXISTS image_url TEXT;
```

4. **Run** (또는 실행) 버튼 누르기  
5. 에러 없이 끝나면 완료

### 2-3. 사진 저장용 창고(버킷) 만들기

1. 왼쪽 메뉴에서 **Storage** 클릭
2. **New bucket** (새 버킷) 클릭
3. **Name**에 `log-images` 라고 **정확히** 입력
4. **Public bucket** 에 체크 (사진을 누구나 볼 수 있게 하려면)
5. **Create bucket** 클릭

여기까지 하면 로그 남길 때 **「사진 추가」** 버튼으로 사진을 올릴 수 있어요.

## 3단계: 나중에 인터넷에 배포할 때

- GitHub에 코드 올리기
- Vercel에서 그 저장소 연결해서 배포
- Vercel에 `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY` 환경 변수 넣기

자세한 건 **DEPLOY.md** 파일을 보면 됩니다.
