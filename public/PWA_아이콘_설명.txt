PWA / 모바일 홈 화면 아이콘

생성된 파일 (픽셀 크기 고정, manifest와 일치)
  - icon-192.png     … 192×192 (any)
  - icon-512.png     … 512×512 (any)
  - icon-maskable-512.png … 512×512 (maskable, 본문은 캔버스의 약 80% 안)
  - apple-touch-icon.png … 180×180 (iOS 홈 화면)
  - icon.png         … 192×192 (favicon 호환용, icon-192와 동일)

마스터 원본
  - scripts/pwa-icon-source.png (정사각 권장)

아이콘 다시 빌드
  npm run icons

원본을 바꾼 뒤에는 위 명령으로 public/*.png 를 다시 생성하고 배포하세요.

── 폰에서 아이콘이 예전 그대로일 때 ──
  • 배포(Vercel 등)에 최신 코드가 올라갔는지 먼저 확인하세요.
  • 아이콘·manifest는 휴대폰이 오래 캐시합니다. 아래를 시도해 보세요.
    - iPhone: 홈 화면에서 앱 아이콘을 삭제 → Safari에서 사이트 열기 → 다시
      「홈 화면에 추가」. (설정 → Safari → 방문 기록 및 웹사이트 데이터 삭제도 도움될 수 있음)
    - Android: Chrome에서 해당 사이트 「사이트 설정 → 저장공간 삭제」 후
      홈 화면 바로가기를 지웠다가 다시 추가.
  • 아이콘을 바꿀 때마다 manifest / 링크의 ?v= 숫자를 올리면 새 이미지를
    받기 쉽습니다. (현재 v=2)
