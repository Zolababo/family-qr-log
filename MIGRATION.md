# Family QR Log — 진행 내역 마이그레이션 (Context 이어가기용)

---

## 0. "새 채팅에서 이어하기" — 매우 상세한 방법 (처음 보시면 여기부터 읽으세요)

### 0-1. 왜 이게 필요한가요?
- Cursor 채팅은 **Context used(사용한 맥락)** 가 쌓이면 100%에 가까워집니다.
- 100%가 되면 **예전 대화 내용을 더 잘 기억하지 못해**, "아까 하던 디자인 작업", "어디까지 했지?" 같은 걸 새 채팅에서는 모릅니다.
- 그래서 **진행 내역을 이 파일(MIGRATION.md)에 적어 두고**, **새 채팅을 열었을 때 이 파일을 같이 보여 주면**, AI가 "지금 프로젝트 상태"와 "지금까지 뭘 했는지"를 바로 알아서 이어서 도와줄 수 있습니다.

---

### 0-2. 새 채팅을 연 다음, 정확히 무엇을 하면 되나요?

**방법 A: @ 로 파일 붙이기 (추천)**

1. Cursor 왼쪽 **채팅 아이콘**을 눌러 **새 채팅**을 엽니다.
2. 채팅 **입력창**(맨 아래에 글 쓰는 칸)을 클릭합니다.
3. **영문 @** 를 입력합니다. (`@` 는 Shift + 2 가 아니라, 키보드에서 @ 문자를 입력하는 겁니다.)
4. @ 를 입력하면 **파일/폴더를 고르는 작은 창**이 뜹니다.
5. 그 창에서 **MIGRATION** 이라고 입력하거나, 목록에서 **MIGRATION.md** 를 클릭해서 선택합니다.
6. 그러면 입력창에 `@MIGRATION.md` 처럼 표시됩니다. (파일이 "붙은" 상태입니다.)
7. 그 다음 **한 줄로 하고 싶은 일**을 씁니다. 예:
   - `이 파일 보고 지금 프로젝트 상태 이어받아서, [하고 싶은 작업] 해줘`
   - `MIGRATION 기준으로 디자인만 조금 더 수정해줘. [예: 버튼 색만 바꿔줘]`
8. **Enter**를 눌러 전송합니다.

**정리**: 새 채팅에서 **@ 입력 → MIGRATION.md 선택 → 하고 싶은 일 적기 → 전송** 하면, AI가 이 파일 내용을 읽고 "진행 내역"을 기준으로 이어서 답합니다.

---

**방법 B: 파일을 열어 둔 상태로 질문하기**

1. Cursor **왼쪽 파일 탐색기**에서 **MIGRATION.md** 를 클릭해서 **에디터에 연다**.
2. **새 채팅**을 연 다음, 채팅 입력창에 예를 들어 이렇게 씁니다:
   - `지금 열어둔 MIGRATION.md 보고, 그 내용 기준으로 이 프로젝트 이어서 도와줘. [그 다음 하고 싶은 일]`
3. 전송합니다.

(이때도 **@MIGRATION.md** 를 입력창에 붙여 주면, AI가 파일 내용을 더 확실히 참고합니다.)

---

### 0-3. "이어서 한다"는 게 구체적으로 무슨 뜻인가요?
- **이 파일에 적힌 내용** = "지금까지 뭘 했는지", "어떤 파일을 건드렸는지", "알려진 이슈는 뭔지" 요약.
- 새 채팅에서 **이 파일을 붙이거나 열어 두고** 질문하면, AI는:
  - "디자인 토큰이 이미 globals.css에 들어가 있고, HomeClient는 그 변수 쓰도록 바꿨구나",
  - "PWA는 icon.png 쓰고, 배포는 Source Control → Commit → Push 구나",
  같은 걸 **이 파일만 보고도** 파악해서, **그 다음 작업(예: 색만 바꾸기, 버튼만 수정)** 을 이어서 할 수 있습니다.

---

### 0-4. 한 줄 요약
- **Context가 많이 쌓였을 때** → **새 채팅**을 연다.
- 새 채팅 **입력창에 @ 치고 → MIGRATION.md 선택** 한 뒤, **"이거 기준으로 이어서 [원하는 작업] 해줘"** 라고 보내면 됩니다.

## 1. 프로젝트 요약

- **스택**: Next.js 16, React 19, Supabase (Auth, DB, Storage), Vercel 배포
- **기능**: 가족용 장소별 로그(QR/버튼), 프로필 사진, 댓글/답글, 캘린더, 메모, PWA, 다국어
- **주요 파일**: `src/app/HomeClient.tsx` (메인 UI), `src/app/globals.css` (전역 스타일), `src/app/translations.ts`

## 2. 최근 완료한 작업 (이어서 할 때 참고)

### 2-1. 디자인 토큰 (globals.css)
- **라이트/따뜻한 테마** 토큰 추가됨:
  - 배경: `--bg-base`, `--bg-card`, `--bg-subtle`
  - 텍스트: `--text-primary`, `--text-secondary`, `--text-caption`
  - 포인트: `--accent`, `--accent-light` (딥 세이지 그린)
  - 장소: `--place-fridge` / `--place-fridge-icon`, `--place-table` / `--place-table-icon`, `--place-toilet` / `--place-toilet-icon`
  - 간격: `--radius-sm`, `--radius-md`, `--radius-lg`
  - 그림자: `--shadow-card`
- **HomeClient.tsx**의 `theme` 객체, `getPlaceChipStyle`, 장소 버튼/칩/로그 버튼/헤더 로고/멤버 칩 등 **하드코딩 색상 → 위 CSS 변수로 교체 완료**.

### 2-2. PWA
- `public/manifest.webmanifest`, `public/icon.png` 사용 중. 사용자가 직접 뱃지 이미지를 `public/icon.png`로 넣어둔 상태.

### 2-3. 기타 이미 적용된 기능
- HEIC 프로필 사진 변환(heic2any), 프로필 탭 시 확대 모달
- 홈에 냉장고/식탁/화장실 아이콘 버튼 → 장소 선택 후 로그 폼
- 댓글/대댓글 들여쓰기(인스타 스타일)
- QR 탭 진입 시 바로 카메라
- 메모: X 버튼 제거, 바깥/스와이프로 닫기
- 장소 기록하기 "해제" 버튼, 촬영/앨범/그리기 가로 스크롤 1줄, 자주 쓰는 문구 UI 정리

## 3. 알려진 이슈 / 남은 작업

- **배포 후 화면이 안 바뀌는 것처럼 보였던 이유**: 예전에는 전역 CSS만 바꿔서 인라인 스타일이 우선했음. **지금은 theme·장소·버튼 등이 CSS 변수로 바뀌어서, 푸시 후 Vercel 배포하면 새 디자인(따뜻한 라이트 + 세이지 그린)이 반영되어야 함.**
- **Context used가 높을 때**: 새 채팅을 열고 **`@MIGRATION.md`** 를 붙여서 이어가기 (섹션 0 참고).
- **스티커**: 로그 `action` 끝에 `\n@@meta:{...}` JSON으로 저장 (위치·스티커 등). **남이 쓴 로그에 스티커**를 붙이려면 Supabase **RLS 정책**이 `logs` 업데이트를 가족 구성원에게 허용하는지 확인 필요 (막히면 정책 수정 또는 별도 테이블).
- **아기 얼굴 인식·성장 타임라인 정렬**: MVP는 참조 사진 등록 + 클라이언트/서버 얼굴 임베딩 비교 같은 단계적 도입이 현실적. 완전 자동은 모델·비용·개인정보 이슈 검토 필요.
- **스크롤 연출**(프로필→장소→로그 순 “펼침”): 요청 있었으나 아직 완전 구현 전. 자연 스크롤·내부 `maxHeight` 제거 등은 진행됨.

## 4. 배포 방법 (상기용)

1. Source Control에서 변경 파일 스테이징(+) → Message에 "배포" 등 입력 → Commit → Sync Changes(Push)
2. Vercel이 `main` 푸시 감지 후 자동 배포. PWA는 필요 시 캐시 삭제 또는 앱 삭제 후 재설치.

## 6. 현재 스냅샷 (2026-03, 새 채팅 이어가기용)

**아키텍처**
- 메인 화면: `src/app/HomeClient.tsx` (탭·로그·캘린더·메모·모달 대부분)
- 컴포넌트: `src/components/layout/AppHeader.tsx`, `BottomTabBar.tsx`, `src/components/home/MemberFilter.tsx`, `PlaceButtons.tsx`, `LogFeed.tsx`
- 스타일: `src/app/globals.css` (토큰, section 카드 제거, 탭/멤버 필터 포커스 등)
- 문자열: `src/app/translations.ts`

**UI/UX (요약)**
- 모바일 우선, 인스타 느낌: 카드 레이어 최소화, 로그 미디어 풀폭(`100vw` + margin 트릭)
- **멤버 필터**: 원형 아바타 + 선택 시 그라데이션 링, 칩 간격·크기 조정
- **하단 탭**: 설정은 모달(이름·프로필·초대·언어·접근성)
- **댓글**: 피드에는 아이콘+개수, 입력은 하단 시트
- **지도 장소 태그**: 로그 작성 시 접기/펼치기, 메타에 `locationName` / `locationUrl` 저장
- **스티커**: 기존 로그에 붙이기(피드에서 스티커 버튼 → 시트). 메타 `stickers` 배열
- **캘린더**: 월 이동, 일별 건수, “오늘의 회상”, **성장 타임라인**(기간 필터 + 미디어 있는 로그)
- **메모**: 우측 패널, 슬라이드 애니메이션
- **촬영**: 카메라 input은 `accept="image/*"` + `capture` (앨범은 `image/*,video/*` 별도)

**배포**
- Vercel: `npx vercel --prod` 로 프로덕션 배포. URL은 배포마다 `*.vercel.app` 형태로 바뀔 수 있음.
